# Plan: clipper.md 以 skill 为基准，仅保留覆盖项

## Context

clipper.md 已经引用了 `.opencode/skills/llm-wiki/SKILL.md` 作为页面模板和规范的来源，但 skill 里的 Page Template（`category:`、`provenance:` 块、`summary:` 必填等）与本项目的实际页面格式不一致。agent 读 skill 后可能按 skill 格式创建页面，导致新旧页面风格分裂。

## 方案

clipper.md 保留现有结构（路由、ingest 流程、概念命名规则等），但：
1. **删除"本项目 vs Skill 的差异汇总"表格**（改用明确的覆盖声明）
2. **新增"Skill 覆盖项"段**，逐条列出本项目不遵循 skill 的规则及替代方案
3. 其余内容不变——skill 是模板/规范的唯一来源，clipper 只管流程和扩展

## 修改文件

- `.opencode/agents/clipper.md` — 替换差异汇总表为覆盖声明

## 具体改动

在 clipper.md 的"前置规则"段之后、"路由"段之前，新增：

```markdown
## Skill 覆盖项

本项目不遵循 `.opencode/skills/llm-wiki/SKILL.md` 中以下规则，以现有实现为准：

| Skill 规则 | 本项目替代 | 原因 |
|-----------|-----------|------|
| `category:` 字段 | 用 `type: concept` / `type: source` | 已有 24 个概念页 + 16 个来源页使用 `type:`，迁移成本高 |
| `summary:` 必填（≤200 chars） | **遵循 skill**，新页面必须填写 | 查询快速模式依赖此字段 |
| `provenance:` 块 | 暂不执行 | wiki 规模小，inferred 比例低，等 200+ 页再启用 |
| `^[inferred]`/`^[ambiguous]` 行内标记 | 暂不执行 | 同上 |
| `## Key Ideas` / `## Open Questions` / `## Sources` 段 | 用领域自定义段（`## 定义`、`## 争议` 等） | 内容更贴合实际 |
| index.md 列表格式 | 用表格（信息更丰富） | 已有表格实现 |
| log.md key=value 格式 | 用叙事体 | 更易读 |
| 时间戳 ISO 8601 | 仅日期 | 够用 |
```

同时删除 clipper.md 末尾的"本项目 vs Skill 的差异汇总"段（已包含在覆盖声明中）。

## 验证

1. 检查 clipper.md 中不再有重复的 Frontmatter 模板
2. 检查覆盖声明与现有页面实际格式一致
3. 确认 agent 读 skill 后会看到覆盖声明并遵循
